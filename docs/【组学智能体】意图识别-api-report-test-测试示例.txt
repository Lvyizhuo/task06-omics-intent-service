# 组学智能体意图识别服务 - 新 /report 接口测试用例

> 本测试针对 PlantCAD2 新统一接口 **POST /report** 验证。
> 新接口特点：
> - 所有 PlantCAD2 任务统一走 `/report` 端点，通过 `type` 字段区分
> - 响应新增 `markdown` 字段，包含前端可渲染的 Markdown 推理报告
> - 位置参数改为 **1-based**（与用户习惯一致）
> - 返回的 `suggested_tasks` / `available_tasks` 中新增 `required_fields` 字段
>
> 注意：实际运行时 DNA 序列会作为参数传入下游 API（PlantCAD2 /report 或 EVO2），
> 如果服务未部署或序列无效，参数提取仍会成功，但接口调用会报错。
> 本测试主要验证**意图识别 + 参数提取 + /report 接口调用 + markdown 输出**是否正常。

## 5. 预期的响应结构

### 5.1 高置信度响应示例（PlantCAD2 任务）
```json
{
  "confidence": "high",
  "task_id": 202,
  "task_name": "变异打分",
  "model": "PlantCAD2",
  "params": {
    "sequence": "ACGTACGTACGT",
    "position": 5,
    "ref_allele": "A",
    "alt_alleles": ["G"],
    "normalize": null,
    "prompt": null,
    "positions": null,
    "numTokens": null,
    "temperature": null,
    "topK": null,
    "topP": null,
    "showLogits": null
  },
  "result": {
    "type": "variant_score",
    "result": {
      "scores": {"G": -0.80, "C": -0.76, "T": 0.77},
      "ref_prob": 0.246,
      "alt_probs": {"G": 0.110, "C": 0.115, "T": 0.529}
    },
    "markdown": "# PlantCAD2 模型推理报告\n\n**来源**：组学智能体 — PlantCAD 模型推理服务\n..."
  },
  "markdown": "# PlantCAD2 模型推理报告\n\n**来源**：组学智能体 — PlantCAD 模型推理服务\n...",
  "guide_message": "已为您完成变异打分。变异打分结果：G: -0.80, C: -0.76, T: 0.77",
  "suggested_tasks": null,
  "available_tasks": null,
  "error": null
}
```

> **验证要点**：
> - ✅ `result` 字段内层包含 `type`、`result`（原始数据）、`markdown`（报告）
> - ✅ 顶级 `markdown` 字段存在且内容与 `result.markdown` 一致
> - ✅ `result.result` 中的 scores 数据与旧接口一致
